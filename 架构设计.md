## 2. 核心模块划分

### 2.1 对话服务 (Dialog Service)
**职责**：处理所有客服对话相关功能
- 多轮对话管理
- 意图识别调用
- 知识库检索
- 上下文维护
- 转人工逻辑

**关键组件**：
```python
DialogManager      # 对话管理器
IntentClassifier   # 意图分类器
KnowledgeRetriever # 知识库检索
ContextManager     # 上下文管理
HandoffHandler     # 转人工处理
```

### 2.2 AI引擎服务 (AI Service)
**职责**：提供AI能力支撑
- 意图识别模型调用
- 文本生成/回复生成
- 情感分析
- 实体提取

**技术方案**：
- 主模型：GPT-4/Claude API
- 备用：本地轻量模型（如需要）
- 意图识别：Fine-tuned 分类模型

### 2.3 平台对接服务 (Platform Service)
**职责**：对接各电商平台
- 淘宝/天猫（千牛API）
- 京东（宙斯API）
- 拼多多（开放平台）

**统一抽象**：
```python
class BasePlatform(ABC):
    @abstractmethod
    async def send_message(self, user_id: str, content: str): ...
    
    @abstractmethod
    async def get_order_info(self, order_id: str): ...
    
    @abstractmethod
    async def get_user_info(self, user_id: str): ...
```

### 2.4 数据分析服务 (Analytics Service)
**职责**：数据统计与监控
- 实时接待统计
- 响应时间分析
- 会话质量评估
- 报表生成

### 2.5 用户服务 (User Service)
**职责**：用户与权限管理
- 客服账号管理
- 角色权限控制
- 店铺绑定管理

## 3. 技术选型说明

| 层级 | 技术 | 选型理由 |
|------|------|----------|
| 后端框架 | FastAPI | 高性能、异步支持、自动API文档 |
| 前端框架 | Vue3 + ElementPlus | 组件丰富、生态成熟 |
| 数据库 | PostgreSQL | 支持JSON、事务强、扩展性好 |
| 缓存 | Redis | 会话存储、热点数据、分布式锁 |
| 消息队列 | RabbitMQ | 可靠消息、延迟队列、削峰填谷 |
| 任务调度 | Celery | 定时任务、异步执行 |
| 搜索 | Elasticsearch | 日志检索、知识库搜索 |
| 文件存储 | MinIO | 兼容S3、私有化部署 |
| 部署 | Docker + Docker Compose | 快速部署、环境一致 |

## 4. 数据流设计

### 4.1 客服对话流程
```
用户消息 → 平台API → API Gateway → 对话服务
                                           ↓
平台API ← 回复消息 ← 回复生成 ← AI引擎 ← 意图识别
                                           ↑
                                    知识库检索 ← PostgreSQL/ES
```

### 4.2 转人工流程
```
触发条件（复杂问题/用户要求）
    ↓
对话服务 → 创建工单 → RabbitMQ → 人工客服系统
    ↓
通知客服 ← WebSocket推送 ← 消息消费
```

### 4.3 数据统计流程
```
对话事件 → 消息队列 → 数据分析服务 → 聚合计算 → Redis/PostgreSQL
                                              ↓
管理后台 ← API查询 ← 实时/历史数据 ← 报表生成
```

## 5. 数据库核心表结构

### 5.1 对话相关
```sql
-- 会话表
conversations (
    id UUID PRIMARY KEY,
    platform VARCHAR(20),      -- 平台类型
    platform_user_id VARCHAR,  -- 平台用户ID
    shop_id VARCHAR,           -- 店铺ID
    status VARCHAR(20),        -- active/closed/pending
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- 消息表
messages (
    id UUID PRIMARY KEY,
    conversation_id UUID REFERENCES conversations,
    direction VARCHAR(10),     -- in/out
    content TEXT,
    msg_type VARCHAR(20),      -- text/image/order
    intent VARCHAR(50),        -- 意图分类
    created_at TIMESTAMP
);
```

### 5.2 知识库相关
```sql
-- 知识库表
knowledge_base (
    id UUID PRIMARY KEY,
    shop_id VARCHAR,
    category VARCHAR(50),
    question TEXT,
    answer TEXT,
    keywords TEXT[],           -- 关键词数组
    hit_count INTEGER DEFAULT 0,
    created_at TIMESTAMP
);
```

### 5.3 统计相关
```sql
-- 每日统计
daily_stats (
    id UUID PRIMARY KEY,
    shop_id VARCHAR,
    date DATE,
    total_conversations INTEGER,
    total_messages INTEGER,
    avg_response_time INTEGER, -- 平均响应时间(秒)
    handoff_count INTEGER,     -- 转人工次数
    created_at TIMESTAMP
);
```

## 7. 安全设计

1. **API安全**
   - JWT Token认证
   - API限流（Rate Limiting）
   - 请求签名验证（平台回调）

2. **数据安全**
   - 敏感数据加密存储
   - 数据库连接SSL
   - 定期备份策略

3. **平台对接安全**
   - OAuth2授权
   - Token自动刷新
   - IP白名单
