# EAMS 核心架构设计

## 系统定位
**多平台电商智能客服中台** - 让商家在一个界面管理所有平台的客户沟通

---

## 核心痛点解决

### 现状问题
- ❌ 商家需要同时打开淘宝千牛、京东咚咚、拼多多商家版等多个窗口
- ❌ 客户信息分散，无法跨平台识别同一用户
- ❌ 重复问题消耗大量人工客服时间
- ❌ 订单/物流查询需要在各平台间切换

### EAMS解决方案
- ✅ **统一收件箱** - 所有平台消息汇聚到一个界面
- ✅ **智能体优先** - 80%常见问题由AI自动处理
- ✅ **用户画像整合** - 跨平台识别用户，聚合购买记录
- ✅ **一站式查询** - 订单、物流、售后信息一键查询

---

## 核心功能模块

### 1. 多平台消息聚合 (Platform Hub)
```
淘宝店铺A ──┐
淘宝店铺B ──┼──► EAMS统一收件箱 ──► AI智能分配/回复
京东店铺 ───┤         ▲
拼多多店铺 ─┘         └──── 人工客服介入
```

**功能点:**
- 各平台Webhook消息接收
- 消息标准化处理
- 平台标识和店铺区分
- 实时消息推送

### 2. 智能客服引擎 (AI Agent Core)

**工作流程:**
```
用户消息 → 意图识别 → 知识库匹配 → 置信度评估
                              ↓
                    ┌─── 高置信度 (>0.8) ───► AI自动回复
                    │
                    └─── 低置信度 (<0.8) ───► 转人工/追问澄清
```

**核心能力:**
- 意图识别（咨询/售后/投诉/议价）
- 知识库RAG检索
- 多轮对话管理
- 情感分析（识别愤怒/满意等情绪）
- 转人工触发机制

### 3. 统一用户画像 (Unified Customer Profile)

**数据整合:**
```
用户手机号/ID ──► 跨平台匹配 ──► 统一画像
                                    │
    ┌───────────────────────────────┼───────────────────────────────┐
    ↓                               ↓                               ↓
淘宝订单记录                    京东订单记录                    拼多多订单记录
├── 购买历史                    ├── 购买历史                    ├── 购买历史
├── 售后记录                    ├── 售后记录                    ├── 售后记录
└── 对话历史                    └── 对话历史                    └── 对话历史
```

**商家视角展示:**
- 客户全平台消费总额
- 最近购买商品（跨平台）
- 历史咨询记录
- 客户标签（VIP/投诉倾向/高价值等）

### 4. 一站式数据查询 (Unified Data Query)

**侧边栏信息面板:**
```

├─────────────────────────────────────┤
```

### 关键交互

**1. 平台切换**
- 左侧平台标签快速筛选
- 不同平台消息用颜色标识
- 支持多平台同时在线

**2. AI辅助提示**
```
用户: "我要退货"
AI建议: "检测到售后意图，建议回复:"
        [1. 确认订单] [2. 询问原因] [3. 发送退货地址]
```

**3. 一键转人工**
- AI置信度低于阈值自动提醒
- 复杂问题一键接管
- 转接时自动传递对话上下文

## 技术实现要点

### 平台对接
| 平台 | 对接方式 | 消息类型 |
|------|---------|---------|
| 淘宝 | 千牛开放平台 | 订单/消息/退款 |
| 京东 | 宙斯开放平台 | 订单/消息/售后 |
| 拼多多 | 多多开放平台 | 订单/消息/退款 |
| 抖音 | 抖店开放平台 | 订单/消息 |

### AI能力
- **意图识别**: 基于BERT的文本分类
- **知识库**: RAG检索增强生成
- **对话管理**: 状态机+上下文记忆
- **情感分析**: 识别用户情绪状态

### 数据同步
- 订单数据定时同步
- 消息实时Webhook接收
- 用户信息跨平台关联
