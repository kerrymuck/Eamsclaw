## 1. 工作流概述

### 1.1 工作流定义

工作流是智能体执行任务的流程定义，包含：
- **节点(Node)**: 工作流中的处理单元
- **边(Edge)**: 节点之间的流转关系
- **条件(Condition)**: 流转的判断逻辑
- **上下文(Context)**: 工作流执行状态

### 1.2 工作流类型

| 类型 | 说明 | 示例 |
|-----|------|------|
| 对话工作流 | 处理用户对话 | 订单查询、售后处理 |
| 数据工作流 | 处理数据任务 | 数据统计、报告生成 |
| 监控工作流 | 处理监控告警 | 实时预警、异常检测 |
| 定时工作流 | 定时执行任务 | 日报生成、数据同步 |

### 1.3 工作流执行模式

```
┌─────────────────────────────────────────────────────────────┐
│                     工作流执行引擎                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────┐    ┌─────────────┐    ┌─────────────┐    │
│   │ 顺序执行    │    │ 并行执行    │    │ 条件分支    │    │
│   │ A → B → C   │    │ A → B,C     │    │ A → {B|C}   │    │
│   └─────────────┘    └─────────────┘    └─────────────┘    │
│                                                             │
│   ┌─────────────┐    ┌─────────────┐    ┌─────────────┐    │
│   │ 循环执行    │    │ 子工作流    │    │ 异步回调    │    │
│   │ A → B → A   │    │ A → [W] → B │    │ A → ○ → B   │    │
│   └─────────────┘    └─────────────┘    └─────────────┘    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 2. 客服智能体核心工作流

### 2.1 主工作流

```yaml
workflow:
  name: customer_service_main
  version: "1.0"
  
  nodes:
    - id: start
      type: start
      next: intent_recognition
      
    - id: intent_recognition
      type: llm
      model: intent_classifier
      prompt: |
        分析用户输入，识别意图类别。
        用户输入: {{user_input}}
        可能的意图: {{intent_list}}
      output:
        - intent
        - confidence
        - slots
      next: intent_decision
      
    - id: intent_decision
      type: condition
      conditions:
        - if: "confidence < 0.6"
          next: clarify_intent
        - if: "intent == 'ORDER_QUERY'"
          next: order_query_flow
        - if: "intent == 'REFUND_REQUEST'"
          next: refund_flow
        - if: "intent == 'LOGISTICS_QUERY'"
          next: logistics_flow
        - if: "intent == 'PRODUCT_INQUIRY'"
          next: product_flow
        - if: "intent == 'COMPLAINT'"
          next: complaint_flow
        - else:
          next: general_chat
          
    - id: clarify_intent
      type: llm
      prompt: |
        用户意图不明确，请礼貌地询问用户具体需求。
        用户输入: {{user_input}}
      output: response
      next: wait_user_input
      
    - id: wait_user_input
      type: wait
      event: user_message
      next: intent_recognition
      
    - id: order_query_flow
      type: subflow
      workflow: order_query
      next: evaluate_response
      
    - id: refund_flow
      type: subflow
      workflow: refund_process
      next: evaluate_response
      
    - id: logistics_flow
      type: subflow
      workflow: logistics_query
      next: evaluate_response
      
    - id: product_flow
      type: subflow
      workflow: product_recommend
      next: evaluate_response
      
    - id: complaint_flow
      type: subflow
      workflow: complaint_handle
      next: evaluate_response
      
    - id: general_chat
      type: llm
      model: chat_model
      prompt: |
        基于知识库回答用户问题。
        用户问题: {{user_input}}
        相关知识: {{retrieved_knowledge}}
      output: response
      next: evaluate_response
      
    - id: evaluate_response
      type: condition
      conditions:
        - if: "user_satisfied"
          next: close_session
        - if: "need_human"
          next: handoff_to_human
        - else:
          next: intent_recognition
          
    - id: handoff_to_human
      type: action
      action: transfer_to_agent
      params:
        reason: {{handoff_reason}}
        context: {{session_context}}
      next: end
      
    - id: close_session
      type: action
      action: close_conversation
      params:
        summary: {{conversation_summary}}
      next: end
      
    - id: end
      type: end
```

## 3. 订单处理工作流

### 3.1 订单查询工作流

```yaml
workflow:
  name: order_query
  
  nodes:
    - id: check_order_id
      type: condition
      conditions:
        - if: "slots.order_id"
          next: fetch_order
        - else:
          next: ask_order_id
          
    - id: ask_order_id
      type: llm
      prompt: |
        询问用户订单号。
        上下文: {{conversation_history}}
      output: response
      next: wait_order_id
      
    - id: wait_order_id
      type: wait
      event: user_message
      next: extract_order_id
      
    - id: extract_order_id
      type: llm
      prompt: |
        从用户输入中提取订单号。
        输入: {{user_input}}
      output:
        - order_id
        - extracted
      next: validate_order_id
      
    - id: validate_order_id
      type: condition
      conditions:
        - if: "extracted"
          next: fetch_order
        - else:
          next: ask_order_id
          
    - id: fetch_order
      type: api
      endpoint: /api/orders/{{order_id}}
      method: GET
      output: order_data
      error_handler: order_not_found
      next: format_response
      
    - id: order_not_found
      type: llm
      prompt: |
        订单未找到，友好地告知用户。
        订单号: {{order_id}}
      output: response
      next: ask_retry
      
    - id: ask_retry
      type: llm
      prompt: |
        询问是否重新输入或查询其他订单。
      output: response
      next: wait_retry_decision
      
    - id: wait_retry_decision
      type: wait
      event: user_message
      next: parse_retry_decision
      
    - id: parse_retry_decision
      type: llm
      prompt: |
        判断用户是想重新输入还是结束查询。
        输入: {{user_input}}
      output: decision
      next: retry_decision
      
    - id: retry_decision
      type: condition
      conditions:
        - if: "decision == 'retry'"
          next: ask_order_id
        - else:
          next: end
          
    - id: format_response
      type: llm
      prompt: |
        格式化订单信息为自然语言回复。
        订单数据: {{order_data}}
      output: response
      next: ask_follow_up
      
    - id: ask_follow_up
      type: llm
      prompt: |
        询问用户是否需要其他帮助。
      output: response
      next: end
```

## 4. 售后处理工作流

### 4.1 退货申请工作流

```yaml
workflow:
  name: refund_process
  
  nodes:
    - id: emotion_analysis
      type: llm
      prompt: |
        分析用户情绪状态。
        用户输入: {{user_input}}
      output:
        - emotion: [satisfied, neutral, anxious, angry, disappointed]
        - intensity: 0-1
      next: emotion_response
      
    - id: emotion_response
      type: condition
      conditions:
        - if: "emotion == 'angry' and intensity > 0.7"
          next: calming_response
        - else:
          next: check_order_info
          
    - id: calming_response
      type: llm
      prompt: |
        用户情绪激动，先安抚情绪。
        情绪: {{emotion}}
      output: response
      next: check_order_info
      
    - id: check_order_info
      type: condition
      conditions:
        - if: "slots.order_id"
          next: fetch_order_for_refund
        - else:
          next: ask_order_info
          
    - id: ask_order_info
      type: llm
      prompt: |
        询问退货所需的订单信息。
      output: response
      next: collect_order_info
      
    - id: collect_order_info
      type: form
      fields:
        - name: order_id
          required: true
          validation: "^\\d{9,20}$"
        - name: product_name
          required: false
      next: fetch_order_for_refund
      
    - id: fetch_order_for_refund
      type: api
      endpoint: /api/orders/{{order_id}}
      output: order_data
      next: check_refund_eligibility
      
    - id: check_refund_eligibility
      type: code
      function: checkRefundRules
      input:
        order: {{order_data}}
        current_time: {{now}}
      output:
        - eligible: boolean
        - reason: string
        - max_refund_amount: number
      next: eligibility_decision
      
    - id: eligibility_decision
      type: condition
      conditions:
        - if: "eligible"
          next: ask_refund_reason
        - else:
          next: explain_ineligible
          
    - id: explain_ineligible
      type: llm
      prompt: |
        解释为什么无法退货。
        原因: {{reason}}
        订单: {{order_data}}
      output: response
      next: offer_alternatives
      
    - id: offer_alternatives
      type: llm
      prompt: |
        提供替代方案（换货、维修等）。
      output: response
      next: end
      
    - id: ask_refund_reason
      type: llm
      prompt: |
        询问退货原因。
      output: response
      next: collect_refund_info
      
    - id: collect_refund_info
      type: form
      fields:
        - name: refund_reason
          type: select
          options:
            - quality_issue: 质量问题
            - not_as_described: 与描述不符
            - changed_mind: 不想要了
            - wrong_item: 发错货
            - other: 其他
          required: true
        - name: refund_detail
          type: text
          required: false
        - name: product_condition
          type: select
          options:
            - unopened: 未拆封
            - opened_unused: 已拆封未使用
            - used: 已使用
          required: true
      next: calculate_refund
      
    - id: calculate_refund
      type: code
      function: calculateRefund
      input:
        order: {{order_data}}
        reason: {{refund_reason}}
        condition: {{product_condition}}
      output:
        - refund_amount
        - shipping_fee_handling
        - deductions
      next: confirm_refund_details
      
    - id: confirm_refund_details
      type: llm
      prompt: |
        生成退款详情确认信息。
        退款金额: {{refund_amount}}
        运费处理: {{shipping_fee_handling}}
        扣减项: {{deductions}}
      output: response
      next: wait_confirmation
      
    - id: wait_confirmation
      type: wait
      event: user_message
      next: parse_confirmation
      
    - id: parse_confirmation
      type: llm
      prompt: |
        判断用户是否确认申请退货。
        输入: {{user_input}}
      output: confirmed
      next: confirmation_decision
      
    - id: confirmation_decision
      type: condition
      conditions:
        - if: "confirmed"
          next: create_refund_ticket
        - else:
          next: ask_modification
          
    - id: ask_modification
      type: llm
      prompt: |
        询问需要修改什么信息。
      output: response
      next: collect_refund_info
      
    - id: create_refund_ticket
      type: api
      endpoint: /api/refunds
      method: POST
      body:
        order_id: {{order_id}}
        reason: {{refund_reason}}
        detail: {{refund_detail}}
        product_condition: {{product_condition}}
        refund_amount: {{refund_amount}}
      output: ticket_data
      next: provide_return_instructions
      
    - id: provide_return_instructions
      type: llm
      prompt: |
        生成退货指引。
        工单号: {{ticket_data.ticket_id}}
        退货地址: {{return_address}}
      output: response
      next: end
```

## 5. 数据监控工作流

### 5.1 实时监控工作流

```yaml
workflow:
  name: real_time_monitoring
  trigger:
    type: scheduled
    interval: 30s
  
  nodes:
    - id: fetch_metrics
      type: api
      endpoint: /api/metrics/realtime
      output:
        - queue_length
        - avg_response_time
        - active_agents
        - satisfaction_score
      next: evaluate_thresholds
      
    - id: evaluate_thresholds
      type: code
      function: evaluateThresholds
      input:
        metrics: {{metrics}}
        thresholds: {{threshold_config}}
      output:
        - alerts: []
      next: process_alerts
      
    - id: process_alerts
      type: condition
      conditions:
        - if: "alerts.length > 0"
          next: send_notifications
        - else:
          next: store_metrics
          
    - id: send_notifications
      type: parallel
      branches:
        - name: dingtalk
          node:
            type: api
            endpoint: /api/notifications/dingtalk
            body: {{alert_message}}
        - name: wechat
          node:
            type: api
            endpoint: /api/notifications/wechat
            body: {{alert_message}}
        - name: sms
          node:
            type: api
            endpoint: /api/notifications/sms
            body: {{alert_message}}
            condition: "alert.severity == 'critical'"
      next: auto_response
      
    - id: auto_response
      type: condition
      conditions:
        - if: "alert.type == 'queue_overflow'"
          next: trigger_overflow_response
        - if: "alert.type == 'response_timeout'"
          next: escalate_priority
        - else:
          next: store_metrics
          
    - id: trigger_overflow_response
      type: api
      endpoint: /api/agents/overflow-mode
      method: POST
      body:
        action: activate
        reason: queue_overflow
      next: store_metrics
      
    - id: escalate_priority
      type: api
      endpoint: /api/tickets/escalate
      method: POST
      body:
        criteria: response_time
        threshold: {{threshold}}
      next: store_metrics
      
    - id: store_metrics
      type: api
      endpoint: /api/metrics/store
      method: POST
      body: {{metrics}}
      next: end
```

## 6. 异常处理工作流

### 6.1 通用异常处理

```yaml
workflow:
  name: error_handler
  
  nodes:
    - id: classify_error
      type: code
      function: classifyError
      input:
        error: {{error}}
      output:
        - error_type: [api_error, llm_error, timeout, validation_error, unknown]
        - severity: [low, medium, high, critical]
        - retryable: boolean
      next: error_decision
      
    - id: error_decision
      type: condition
      conditions:
        - if: "retryable and retry_count < 3"
          next: retry_with_backoff
        - if: "severity == 'critical'"
          next: escalate_error
        - else:
          next: graceful_degradation
          
    - id: retry_with_backoff
      type: code
      function: calculateBackoff
      input:
        retry_count: {{retry_count}}
      output: delay_ms
      next: wait_before_retry
      
    - id: wait_before_retry
      type: sleep
      duration: {{delay_ms}}
      next: retry_original_node
      
    - id: retry_original_node
      type: jump
      target: {{failed_node_id}}
      next: end
      
    - id: escalate_error
      type: api
      endpoint: /api/alerts/critical
      method: POST
      body:
        error: {{error}}
        context: {{workflow_context}}
        timestamp: {{now}}
      next: notify_admin
      
    - id: notify_admin
      type: api
```
