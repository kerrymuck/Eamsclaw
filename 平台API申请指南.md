## 1. 淘宝/天猫（千牛开放平台）

### 1.1 需要申请的API权限

| API名称 | API路径 | 用途 |
|---------|---------|------|
| 接收消息 | `taobao.tmc.message.consume` | 接收买家消息 |
| 发送消息 | `taobao.openim.customerservice.msg.send` | 发送客服回复 |
| 订单查询 | `taobao.trade.fullinfo.get` | 查询订单详情 |
| 用户信息 | `taobao.user.buyer.get` | 获取买家信息 |
| 商品查询 | `taobao.item.seller.get` | 获取商品信息 |
| 物流查询 | `taobao.logistics.trace.search` | 查询物流轨迹 |

### 1.2 申请流程

```
1. 注册企业账号
   ↓
2. 创建应用（选择"商家自研"或"服务商"）
   ↓
3. 提交资质审核（营业执照等）
   ↓
4. 申请API权限包
   ↓
5. 等待审核（3-5工作日）
   ↓
6. 获取AppKey/AppSecret
   ↓
7. 店铺授权绑定
```

### 1.3 所需材料

- [ ] 企业营业执照（彩色扫描件）
- [ ] 法人身份证正反面
- [ ] 应用功能说明书
- [ ] 数据安全承诺书
- [ ] 服务器域名备案信息

### 1.4 申请地址
- 开放平台：https://open.taobao.com/
- 千牛文档：https://open.taobao.com/doc.htm

## 4. 时间风险评估

### 4.1 正常流程时间线

```
今天（3月16日）提交申请
        ↓
3月17-18日 平台审核中
        ↓
3月19-20日 审核通过，获取权限
        ↓
3月20日起 可开始平台对接开发
```

### 4.2 风险点

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 审核不通过 | 中 | 高 | 准备详细的功能说明文档 |
| 审核时间延长 | 中 | 中 | 同时申请多个平台，并行处理 |
| 需要补充材料 | 高 | 中 | 提前准备所有材料 |
| 企业资质问题 | 低 | 高 | 确认营业执照在有效期内 |

## 5. 备选方案（如正式申请来不及）

### 5.1 沙箱环境测试

各平台都提供沙箱/测试环境：

| 平台 | 沙箱环境 | 可用性 |
|------|----------|--------|
| 淘宝 | 有 | ✅ 可用 |
| 京东 | 有 | ✅ 可用 |
| 拼多多 | 有限 | ⚠️ 部分可用 |

**建议**：
- 先用沙箱环境完成开发
- 正式权限下来后直接切换

### 5.2 测试账号方案

- 使用平台提供的测试店铺
- 模拟真实对话场景
- 验证核心功能逻辑

### 5.3 最小可行方案（MVP）

如果部分平台申请延迟：

| 阶段 | 支持平台 | 功能 |
|------|----------|------|
| MVP v1 | 淘宝（最先下来） | 基础对话 + 知识库 |
| MVP v2 | 淘宝 + 京东 | 多平台 + 订单查询 |
| MVP v3 | 全平台 | 完整功能 |
